# Contributing to YC-Factory

欢迎大家来完善！🎉

YC-Factory 的核心哲学是：**简单、可组合、可扩展**。

---

## 什么样的 PR 值得被合并？

### ✅ 欢迎的贡献

**1. 新 Agent 类型**
- 新的角色定义（Sales Agent、HR Agent、Data Agent...）
- 现有 Agent 的能力增强
- 新场景的 startup 模板

**2. Protocol 改进**
- 更清晰的消息格式
- 更好的状态同步机制
- 新的 task flow 模式

**3. 文档 & 示例**
- 更清晰的 README
- 新的使用场景文档
- 教程、案例分析

**4. 框架集成**
- 支持新的多 Agent 框架
- 适配 OpenClaw / Claude Code 的新特性

**5. Bug 修复 & 体验优化**
- 脚本错误修复
- CLI 体验改进

---

### ❌ 不欢迎的贡献

**1. 过度工程化**
- 加一堆抽象层但没有实际价值
- 为了"可扩展性"加复杂度
- YAGNI（你不需要它）

**2. 破坏性改动**
- 改核心概念但不向前兼容
- 删现有功能（除非明显有问题）

**3. 与 YC 哲学相悖**
- 加"KPI 驱动"的东西
- 加重的流程管控机制

---

## 核心原则

> **Keep it simple. Keep it modular. Keep it YC.**

YC-Factory 不是要做一个"完整的企业级创业平台"。它是一个**最小可行的创业操作系统**——足够简单让任何人都能改，足够模块化让任何框架都能接。

如果你的 PR 让系统变得更复杂，先问自己：能不能不加？

---

## 提 PR 之前

1. 先开 issue 讨论大的改动
2. PR 要有清晰的描述：改了什么、为什么改
3. 保持代码风格一致
4. 提交信息用中文或英文都可以

---

# Contributing to YC-Factory (English)

Welcome! 🎉

YC-Factory's core philosophy: **Simple. Modular. Extensible.**

---

## What PRs are worth merging?

### ✅ Welcome contributions

**1. New Agent Types**
- New role definitions (Sales Agent, HR Agent, Data Agent...)
- Enhanced existing agent capabilities
- New startup templates for specific scenarios

**2. Protocol Improvements**
- Cleaner message formats
- Better state sync mechanisms
- New task flow patterns

**3. Docs & Examples**
- Cleaner README
- New usage scenario docs
- Tutorials, case studies

**4. Framework Integrations**
- Support for new multi-agent frameworks
- Adapting to new OpenClaw / Claude Code features

**5. Bug Fixes & UX**
- Script error fixes
- CLI experience improvements

---

### ❌ Not welcome

**1. Over-engineering**
- Abstract layers with no real value
- Adding complexity for "extensibility" that isn't needed
- YAGNI (You Aren't Gonna Need It)

**2. Breaking Changes**
- Changing core concepts without backward compatibility
- Removing existing features (unless obviously broken)

**3. Against YC Philosophy**
- Adding "KPI-driven" mechanisms
- Heavy process control

---

## Core Principle

> **Keep it simple. Keep it modular. Keep it YC.**

YC-Factory is NOT trying to be a "complete enterprise startup platform." It's a **minimum viable startup OS** — simple enough for anyone to modify, modular enough for any framework to plug into.

If your PR adds complexity, ask yourself: can we not?

---

## Before Submitting a PR

1. Open an issue first for big changes
2. PR should have a clear description: what changed and why
3. Keep code style consistent
4. Commit messages in Chinese or English — either is fine
